“荔枝”来了！新官网有木有？

作者: dron  发表日期: 2011-08-22

荔枝项目的主要目标是提高网站的易用性，因启动该项目时，荔枝正是时下的水果，故项目起名荔枝。荔枝项目对于官网的改进点如下：

API 页面更丰富了

以树状菜单展示所有的 API 接口，向用户表达了很强的层次感，帮助用户加深对 Tangram API 的认识；API 详情页按实际使用顺序分别展示了语法、Options 参数、方法、事件、插件以及 Demo 几个分块，分块可以分别折叠或展开；如果方法、事件和插件等另有更详细的说明，可以再单独展开查看。

每个 API 项均带有 Demo，Demo 提供控制台方便用户对其进行相应的操作，控制台的操作结合了 API 提供的各种方法、事件、插件等，使用户更直观地看到各 API 的表现。

去看看一个例子

优化后的自定义 Tangram 页面

同样是树状菜单，不同的是，操作响应速度对比老版在非 IE 下提升了近 10 倍，IE6 则更加给力，提升了有 170 倍左右；在表现上，树形菜单的勾选、不选以及半选的状态，比老版的黄框、绿框等更容易让人理解。

老版的“我的选择”栏目一直处于不可用状态，这次也顺便改了，不过不同的是，这次是“生成文本”和“载入选择”功能，用户可以保留生成的文本信息以便下次使用，同时，这些文本信息也将与用户自己开发的工具更方便的互通，今后计划对自定义 Tangram 页面开放一些交互接口，“我的选择”在今后将会发挥更多作用。

去看看自定义 Tangram 页面

API、Custom 服务化

此次 API、Custom（原 CodeSearch）一改原来的做法，将数据和页面完全剥离开，为的是给今后工具类或第三方开发的接入做铺垫，API、Custom 将形成 Tangram 的两大子服务，逐步渗透到使用 Tangram 的各种场合当中。

顶部导航菜单通用化

顶部导航菜单做成组件，支持无限子级，方便数据的维护，同时也修复了老版顶部导航菜单在 IE6 下的多个 bug。

搜索优化

将各种类型的文章、API 的详细说明也纳入到搜索的索引当中，搜索能力更强了，能找的东西更多了。

其它一些易用性改进

荔枝项目还对官网进行了多处细节的易用性改进，留待大家慢慢发现吧。
